Javascript Funktioner

Här finns det material du behöver för att klara Uppgift 3 - Javascript Funktioner

Om funktioner

Varför funktioner? Olika typer av funktioner? Hur skriver man en funktion?

Olika typer av funktioner

I Javascript kan man göra väldigt mycket med funktioner. Man kan exmpelvis köra en funktion som argument till en annan funktion. Vi kommer inte kunan täcka alla deltaljer om funktioner i kursen, utan endast skrapa på ytan. På det stora hela kan man dela in funktioner i Javascript som:

  • Namgivna funktioner (eng. named functions)
  • Anonyma funktioner (eng. anonymous function)
  • Immediately invoked functions, förkortat IIFE (hittar ingen översättning)

I kursen kommer ni primärt i kontakt med namngivna funktioner, lite om anonyma. IIFE är lite överkurs, men är bra om du vill fördjupa dig.

Strukturen för en funktion

Även om funktioner kan skrivas på lite olika sätt i Javascript finns dessa beståndsdelar med mer eller mindre. Ordet function talar om för Javascript att den ska tolkas som en funktion. Om funktionen är en namngiven funktion (named function) så anges dess namn. En funktion kan ha inparametrar, men behöver inte ha det. Detsamma är det med utparametern, en funktion kan returnera ett svar som i det här fallet. Eller utelämna det, då utförs endast de rader som är innan för kodplocket. Om man vill köra en funktion anropar man det med dess namn + eventuella inparametrar. Om funktionen returnerar ett värde måste man spara det i en variabel.

Funktioner är bra då de kan ta inparametrar och ge ett svar med return. Funktioner spelar en stor nytta då de blir återanvändbara och man inte behöver upprepa sin kod.

För namngivna och anononyma funktioner finns det två skrivsätt som ni kommer att jobba med i Uppgift 3:

  • Funktions deklaration (eng. function declaration
  • Funktionsuttryck (eng. function expression)

Funktions deklaration

I Javascript brukar man skilja på att skriva en funktions deklaration. Detta skrivsätt är det som ni bekantat er med i Programmering 1. Funktionen har ett namn och körs när den anropas. Funktionen kan returnera ett värde eller endast utföra de isntruktioner som står i kodblocket

Funktions uttryck

I Javascript används även ett annat sätt att skriva funktioner. Det kallas funktionsuttryck eller function expression på engelska. I detta fall behöver inte funktionen ha ett namn, d.v.s den är anonym, utan funktionen tilldelas till en variabel. Man anropar sedan variabeln för att köra funtkionen.

Funktions deklaration eller funktionsuttryck?

Att välja mellan funktions deklaratation och funktionsuttryck är oftast en stilfråga, eller hur man lärt sig från början. Dock finns det några skillnader. För en funktionsdeklaration kan man köra funktionen tidigare i scriptet, innan funktionen deklarerats. Det kallas för hoisting. Funktionsuttryck kan inte hoistas, d.v.s man måste anropa funktionen efter att variabel har tilldelats funktionen. En annan skillnad är att i ett funktionsuttryck sparas funktionen i en variabel. Denna variabel blir också på så sätt möjlig att spara över.

Immediately invoke function (IIFE)

Dessa funktioner anropar man utan körs direkt när de stöts på i koden.
