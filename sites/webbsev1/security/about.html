Säkerhet

När man arbetar med webbutveckling och i synnerhet webbserverprogrammering bör man alltid från början utveckla ett säkerhetstänk och skriva kod som förebygger mot eventuella attacker. Till skillnad från andra plattformar och utvecklingsmiljöer är webben extra sårbar då webbplatser/webbaplikationer är åtkomliga för hela jordklotets nätanvändare.
Det största felet man kan tänka som webbutvecklare är "Varför vill någon hacka min sida?". Tyvärr finns det både hackare som medvetet vill komma åt eller förstöra din data. Likväl finns det användare som av misstag kan göra saker som agerar skadligt på webbplatsen/webbapplikationen. Ett annat problem är s.k botar som per automatik sprider spam eller överbelastar ett system.

Som webbutvecklare bör du istället tänka - LITA ALDRIG PÅ NÅGON- och utveckla webbplatsen eller webbapplikationen därefter.

PHP är ett dynamiskt och flexibelt språk som lätt kan missbrukas om man inte känner till de potentiella säkerhetsluckorna som kan uppstå. Exempelvis hur skyddar man sig från skadlig kod i som matas in i formulärfält, kryptering av lösenord eller att en session kan kapas?
Det är viktigt att känna till de vanligaste säkerhetsproblemen som kan uppstå och hur man förhindrar eller minskar chansen för att dessa skall uppstå. Vissa problem är vanligare än andra. Nedan kan du lära dig mer om de 10 största säkerhetshoten enligt organisationen OWASPs lista 2017 för att få en fördjupning inom säkerhetsproblematik

Säkerhet inom webbserverprogrammering är ett stort område. I den här kursen kommer vi att begränsa oss till att implementera försvarsåtgärder kring:

Filtrering av indata och escaping av utdata SQL-injektioner Kryptering av lösenord

OWASP Top 10

OWASP (Open Web Application Security Project) är en ideell organisation som arbetar med att skapa gratis kunskap, metoder, dokument, verktyg och tekniker för att hindra sabotage på webbsajter.Enligt OWASP så är de 10 vanligaste säkerhetsriskerna år 2017:

Injection Attacks Broken Authentication Sensitive Data Exposure XML External Entities Broken Access Control Security Misconfiguration Cross-Site Scripting (XSS) Insecure Deserialization Using Components With Known Vulnerabilities Insufficient Logging and Monitoring

Länkarna går till videor kring respektive säkerhetshot. En bra och tydlig förklaring om vilka hot som är mest aktuella idag.
