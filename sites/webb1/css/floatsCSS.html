Floats

Hittils har vi tittat på hur olika CSS egenskaper kan ändra utseendet på text, ge olika färger på förgrund och bakgrund men också hur man kan ändra de olika boxarnas inner- och yttermarginal, samt kantlinje. Vi har endast formaterat våra element såsom de befunnits sig i html-dokuments normala flöde.
I detta avsnitt ska vi titta på hur vi kan positionera våra element enligt andra önskemål. Dock innebär det att elementen måste frångås det normala flödet hur elementen visas på en sida.

Vad menas med ett dokuments normala flöde?

Per default visas elementen uppifrån och ner efter hur de är specifierade i html-dokumentet. Det finns dock en skillnad mellan olika typer av element (se avsnitt Boxmodellen).
Block-element börjar alltid på ny rad. De tar samtidigt upp 100% bredd av sitt föräldraelement och höjden avgörs av elementens innehåll. Inline element ordnas efter varandra (ej ny rad) och har bredden och höjden av sitt innehåll. Vi demomonstrerar med ett litet exempel på det normala flödet där ingen positionering är gjord i stilmallen.

Jag är ett block-level element. Närliggande 
block-element är efter mig.
</p>

defineras av mitt innehåll. Min totala bredd/höjd enligt
boxmodellen är content + padding + border width/height.
</p>

Vi block-element har mellanrum som är definerad av en 
margin
</p>

Detta är ett div element och är också ett 
block-element
</div>

Att avända oss av Floats

Med hjälp av CSS egenskapen float kan ett element "plockas" ur det vanliga flödet och förflyttas enligt önskemål. Med värdet left förflyttas elementet så långt den kan till vänster i föräldralemenetet eller till höger med värdet right. Defaultvärdet till float är none, d.v.s att element ingår i det normala flödet. Vi visar med ett exempel med ett p-element innehållande både text och bild.

Ingen float - normala flödet

Bilden är ett inline-element där texten fortsätter på samma rad. Eftersom bilden är högre än texten blir det ett tomt uttryme ovan texten till höger om bilden.

Detta är default-läget och samma sak som att sätta egenskapen float till none .

Bilden till vänster

Bilden går så långt till vänster som den kan med float satt till left. Texten "wrappas" omrking bilden. För att få lite mellrum mellan text och bild är margin satt till 1em. 

img {
    float: left;
    font-size: .7em;
    margin: 1em;
}

Kuriosa: Analogi om floats


Ett element har har tagits från sin normala position men fortsätter påverka sin omgivning. På samma sätt som en ö omgiven av ström kan du se ett element med egenskapen float som en ö i strömmande vatten. Den är inte i flödet men strömmen runtomkring påverkas av den.

Bilden till höger

Vi får samma effekt om vi sätter float till right, d.v.s till bilden flyter till höger och texten runtomkring. 

img {
        float: right;
        font-size: .7em;
        margin: 1em;
}

Att använda Clear

Element som justerats med float ordnas i sidled till höger eller till vänster. För att komma tillbaka till normala födet kan man använda clear på elementet som kommer efter det som satts till float: left eller float: right. Nedan är ett exempel där bilden har satts till float: left; och underrukbriken sätts till clear: left; för att komma till ny rad, d.v.s återvända till normalflödet.

Till egenskapen clear finns förutom left, även right och both. Om flera element är satta till float både till höger och vänster kan värdetboth användas.

Problem med floats

Floats har funnits länge inom CSS och det är de som har tagit om hand hur man vill placera olika sektioner, d.v.s layout. Dock kan det dyka upp märkliga beteenden som det är bra att känna till och hur man kan komma tillrätta med dem.

Vanligt försöker alla blockelement att sätta en höjd som täcker in allt innehåll som finns i dem - om inte height har angetts. När vi använder float på ett element "plockas" det ur det normala flödet och räknas inte med som en del i föräldraelementet. Det här blir ett problemt som vi kan se i följande exempel.

Kantlinjen visar föräldraelementet innehållande texten och bilden. När bilden flyter räknas inte elementets mått med och kommer utanför föräldraelementet. Detta problem kallas för "the great collapse" och kan åtgärdas med det s.k clearfix-hack. Det clearfix-hack som rekommenderas som bästa lösning till de flesta siutaioner är följande: 

.clearfix::after
{
        content: "";
        clear: both;
        display: table;
}

Denna teknik skapar ett s.k pseudoelement efter element som man anger som class="clearfix" med en clear: both. Om du vill veta mer om detaljer läs mer på denna länk.

Genom att sätta class="clearfix" på vårt föräldraelement, i detta fall <p>-elementet tar nu bilden plats i elementet och hamnar inte utanför.

Kuriosa

Den klassiska metoden att skapa layout för webbsidor är att använda sig av floats. Idag finns det modernare alternativ somm CSS Flexbox och Grids, men floats är fortfarande vanligt förekommande och det är bra att kunna tillämpa den klassiska metoden för CSS layout.

Övningar

1. Använda float på bilder med text

Skapa et html-dokument och en tillhörande stilmall (css). I <body> skapa en <div> som innehåller två bilder och ett par stycken text. För att lätt använda dig av bilder kan du länka bilder från en tjänst som heter Lorem Picsum. Prova att använda float: left och float: right på vardera bild så att text och bild visas snyggt tillsammans. Tycker du att en del text ska hamna på ny rad? Prova då att omsluta den texten med en <div> och använd sedan clear: both på det elementet.

2. Enkelt bildgalleri

Skapa et html-dokument och en tillhörande stilmall (css). I <body> skapa en <div> som innehåller sex stycken bilder. Målet är att få bilder efter varandra på två rader med tre bilder i vardera. Använd tjänsten Lorem Picsum här - eller om du hellre vill använda dina egna bilder. Varje bild ska sättas till float: left och ha en width: 33.33%, d.v.s en en tredjedel var av ytan. Tänk på att du måste använda dig av "clearfix-hack" för att de flytande elementen ska befinna sig inom föräldraelementet.
