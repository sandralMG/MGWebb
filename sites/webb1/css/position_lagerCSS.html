Positionering och lager

Vi har lärt oss att element/boxar ordnas utifrån det ursprunliga normala flödet. Med egenskapen float kan element frångå det normala flödet och flyta till vänster eller höger. Dock är relationen mellan dem i layoutflödet bevarad.

Genom positionering och egenskapen position kan elementen plockas ur det normala flödet och placera dem exakt där vi vill. Man brukar skilja mellan aboslut och relativ positionering.

position: static

Om ingen positionering anges är static det som är defaultvärdet.. D.v.s elementet visas i det normala layout flödet.

position: relative

Elementet/boxen kan förflyttas med relativt till positionen i det normala flödet, d.v.s static. Utifrån det kan man ange positionsangivelserna top, bottom, right, left. Utrymmet från det elementet förflyttas blir tomrum.

position: absolute

Absolut positionering innebär till skillnad från realtiv positionering att elementet/boxen tas helt ur det normala flödet. Det efterlämnar inget tomrum och andra element kan ta upp utrymmet. Absolutpositionering använder på samma sätt positionsangivelserna top, bottom, right, left. I detta fall kan även elementet placeras ovanpå varandra och dölja det underliggande innehållet. Vilken ordning som visas stys av z-index som vi förklarar närmare under Lager

position: fixed

Precis som med absolute tas elementet ur det normala flödet. Positionsangivelserna utgår från webbläsarfönstret. När webbläsaren scrollar så ligger elementet kvar i sin positon. Denna teknik används ofta som en fixerad meny vid toppen eller sidfot vid botten av sidan.

Lager med z-index

Om bilder hamnar på samam position hamnar de som lager ovanpå i en bestämd ordning. Den ordningen kan man justera med hjälp av att sätt z-index på det element man vill ska ordna framåt mot skärmen eller bakåt. Ett positivare z-index kommer gör att elementet ligger över, medan ett lägre hamnar under.
