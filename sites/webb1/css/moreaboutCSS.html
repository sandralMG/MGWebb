Mer om CSS

När man arbetar med stilmallar behöver man ha några viktigt koncept i åtanke för att kunna förstå och använda sig av CSS regler på rätt sätt.

Arv, föräldrar och barn

Vad är arv?
Arv innebär att CSS-egenskaper förs vidare till underliggande element.
Vi tittar på HTML-koden för att se hur arv fungerar. Lägg märke till att en del element ligger inuti ett element.
Exempelvis ligger hela sidan i <body>-elementet.
<h1> och <nav>-elementet ligger inuti <header>-elementet.
På samma sätt har även <article>, <p> och <footer> underelement. 

<!DOCTYPE html>
<html>
<head>
</head>
<body>
     <div id="wrapper">                                  
        <header>
            <h1>En huvudrubrik</h1>
            <nav>En navigationsmeny</nav>
        </header>
        <article>
            <h2>En underrubrik</h2>
            <p>
              Lite text och en <a href="#">länk</a> . <em>Länkar är bra att ha</em>. En till <a href="#">länk</a>.
            </p>
        </article>
        <footer>
            <a href="#">Kontakt</a>
            <a href="#">Här finns vi</a>
        </footer>
    </div>
</body>
</html>

Vi kan även visa hur våra html-element är ordnade enligt en struktur enligt bilden. Bilden är till att vi ska förstå hur arv fungerar i CSS.

body {
 font-family: Arial;
 font-size: 18px;                                
}

Om vi använder ovanstående CSS-regel kommer den alltså att verka på alla underordnade element till <body>. Vi säger att alla element inuti <body> ärver egenskaperna till CSS-regeln.

Om vi endast skulle ange en regel på footer skulle endast de underornade elementen till <footer> att påverkas. 

footer {
         font-family: Verdana;
         color: blue;                              
        }

Vi säger att alla element inuti <footer> ärver egenskaperna för CSS-regeln.

Föräldrar och barn

Den bild med förgreningar till alla element kan vi kalla för dokumentträd. Vi kan åskadlidgöra det som ett familjeträd med relationer emellan.
Ett underordnat element blir således barn till det överordnade föräldern. I engelska termer parent och child. Det här är viktiga begrepp att känna till inom webbutveckling.

Varför är arv användbart?

Det har skapats för att underlätta för webbutvecklare.

  • Annars skulle vi behöva ange egenskaper för alla underliggande element.
  • CSS-filerna skulle blir stora och svåra att underhålla.

Ärvs alla egenskaper i CSS?

NEJ.
Det finns vissa egenskaper som inte ärvs - exempelvis border är inte en praktisk egenskap att ärvas. Om vi sätter en border på p-elementet vill vi ju inte att varje del av texten vi markerar ska ha kanter.

Rent allmänt: De egenskaper som gör jobbet lättare. Endast egenskaperna för:

  • Text
  • Listor
  • Färg

Egenskapen font-size har lite särskilda regler som vi återkommer till

Konflikt i stillmallsregler

Vad händer om ett element får en viss regel fast på två olika ställen i stilmallskoden? Ett html-dokument kan även referera till flera stilmallar.
Det finns särskilda regler som tar hänsyn till det - men vi kommer inte gå in på alla dessa i denna kurs. En regel som dock är viktig att känna till som gäller inom em och samma stilmall:

Regelordning

Det som kommer senare i CSS-dokumentet är den regel som gäller. 

p {
color: red;
color: green;
color: blue;                           
}

I det här exemplet kommer alltså texten i paragrafen få en blå färg - efter det "skrivs över" de tidigare. Eller det engelska begreppet "overrides" .
Detta är dock praktiskt när man vill skapa en "fallback" om den nya CSS-tekniken man vill använda inte stöds fultl ut i alla webbläsare. 

h1 {
/*Fallback first*/
border: 25px solid #eee;
/*Newer technique */
border-image: url(fancy.png) 55 fill / 55px / 25px;                                       
}

D.v.s om webbläsaren har support för border-image kommer den använda sig av egenskapen border som fallback.
